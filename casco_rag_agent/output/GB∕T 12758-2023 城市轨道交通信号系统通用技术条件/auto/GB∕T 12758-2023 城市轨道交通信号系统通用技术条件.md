# 城市轨道交通信号系统通用技术条件

# 1范围

本件规定了城市轨道交通信号系统的总体要求、系统及系统要求、车辆基地信号系统、机界面、电源、接口、RAMS、系统性能、电磁兼容性与防护的要求和适用的环境条件。

本件适用于城市轨道交通地铁、轻轨、单轨、中低速磁浮、市域快速轨道、自动导向轨道系统。

# 3术语和定义

GB/T50833界定的以及下列术语和定义适于本件。

# 城市轨道交通信号 urban rail transit signal

应用于城市轨道交通系统，人工或自动实现车指挥、列车运控制、安全间隔控制和管理的技术、措施的总称。

3.2

# 调度集中centralized traffic control

在调度中心内集中显示某一区段内车站、区间、信号、道岔、轨道区段的状态以及列车运行位置，由调度员集中控制区段内的信号和道岔，并指挥列车运的系统设备。

3.3

# 列车自动监控 automatic train supervision

动实现车指挥控制、列车运监视和管理等技术的总称。

3.4

# 列车自动防护 automatic train protection

实现列车运间隔、超速防护、进路安全和车门等动控制以及其他与相关专业联动共同实现列车运行安全防护技术的总称。

3.5

# 列车自动运行 automatic train operation

动控制列车实现启动、速度调整、定点停车、车门开闭功能以及其他分配给司机的驾驶功能等的技术总称。

3.6

联锁interlocking道岔、区段、信号机等按定的规则和条件建的相互关联、制约的关系。

3.7

闭塞 block信号或凭证保证运列车之间保持安全隔离的技术法。

3.8

固定闭塞 fixed block

线路被预先设定若个固定的区段，前后列车间的最安全间隔距离为或多个区段的闭塞式。

3.9

# 准移动闭塞 quasi-moving block

线路被预先设定若干个固定的区段，后行列车以前行列车所占用区段的入口为追踪目标点，前后列车间的最小安全间隔距离为后列车安全制动距离加上安全裕量的闭塞式。

3.10

# 移动闭塞 moving block

线路不划分固定的区段，后列车以前行列车所占线路的安全距离末端为追踪目标点，前后列车间的最小安全间隔距离为后列车安全制动距离加上安全裕量的闭塞式。

3.11

# 进路闭塞 route block

列车运间隔为进路始端信号机相邻下架顺向信号机之间的闭塞法。

[来源:GB/T50833—2012,8.3.7]

3.12

# 站间闭塞 inter-stationblock

列车运间隔为相邻两座车站出站信号机之间的闭塞法。

[来源：GB/T50833—2012,8.3.6]

3.13

# 全自动运行 fully automatic operation

基于计算机、通信、控制和系统集成等技术，由信号、车辆、综合监控、通信、站台屏蔽门等与列车运相关的设备联动实现列车无司乘人员干预的全过程动运。

3.14

# 行车间隔 running interval

根据运计划，在线路上任意点，同向连续运列车间的时间间隔。

3.15

通过能力 carrying capacity

在定车辆类型和定的车组织法的条件下，信号系统在单位时间内所能控制同向通过或接发车的最多列车对数。

3.16

# 折返能 turn-back capacity

同折返车站单位时内能够连续完成折返作业的最列车对数。

# 5总体要求

5.1 城市轨道交通信号系统（以下简称信号系统）应安全可靠、经济合理，各系统设备应能协同作，完成列车运控制的整体标。应具有车安全防护、提运效率、提列车运动化平的作用。

5.2 信号系统应具有可靠性、可性。涉及车安全的系统、设备、电路应符合故障-安全原则。

5.3 信号系统应适应新技术的应，可实现不同系统或系统设备的互与协同作。

5.4 信号系统应根据运营需求确定系统的构成与规模，并应满线路延伸扩展的需求。

5.5信号系统应满城市轨道交通不同运量和车密度的运营需要。

5.6 全封闭线路的城市轨道交通系统应配备ATP系统;线路部分封闭的城市轨道交通系统，应根据车间隔、列车运速度、线路封闭状态等运营条件采取相应的技术段进列车运安全防护。

5.7 设有平交道口的非封闭线路应具有道口防护功能。信号系统可独设置道口信号设备，也可与市政道路交通信号控制系统合设道口防护设备。独设置的道口信号设备应符合GB10493和GB10494的规定。当与市政道路交通信号控制系统合设道口防护设备时，宜采用与市政道路交通相一致的安全措施。若配置ATP子系统，道口防护设备宜与ATP系统联动。

5.8 车指挥控制中(以下简称控制中)应实现本线运列车的统指挥调度，可实现线列车运的统指挥调度，可实现轨道交通线间线路的协同指挥。

5.9 城市轨道交通线路应能组织独运。具有互联互通需求的线路，信号系统应满运营需求，且应符合下列规定：

a）城市轨道交通不同闭塞制式信号系统的跨线运宜采用兼容式实现；b）同闭塞制式信号系统与互联互通需求相关的系统设计原则、系统功能分配、

5.10 信号系统应实现列车安全运，并在系统最不利条件下，当前列车处于紧急停车时，后续列车应能安全停车。

5.11 信号系统应能适应不同编组长度列车的混合运，可持列车在线联挂和解编。

5.12 信号系统的数据通信络应采冗余的络结构或冗余的传输通道，车地线通信络宜采城市轨道交通专用频段。

5.13 信号系统应具有设备监测和报警能，宜设置集中的监测报警系统。各系统应具备检、诊断的维护管理功能。

5.14 当信号系统设备故障时，应具有降级运模式和应急措施。系统宜采取有效的技术措施提升降级运行的安全和效率。

5.15 信号系统的车载设备不应超出车辆限界，信号系统的地设备不应侵设备限界。

5.16 信号系统的设备应符合城市轨道交通使环境与运条件。设于架线路或地线路的信号谈备应与城市景观相协调。

5.17 信号系统的应应结合相配套的系统操作和运营管理措施。

# 6 系统要求

# 6.1 系统分类

6.1.1按闭塞制式划分，可分为固定闭塞制式、准移动闭塞制式和移动闭塞制式。  
6.1.2 按地面设备向车载设备传递信息的连续性划分，可分为在固定地点传递信息的点式、沿线路连续传递信息的连续式。  
6.1.3 按信号系统控制模式曲线划分，可分为阶梯式和速度-距离模式曲线。

# 6.2 系统构成

6.2.1 信号系统可由车指挥控制系统或具有车指挥控制功能的设备、ATP系统、ATO系统、联锁系统或具有联锁功能的设备、数据通信系统、维护监测系统和信号机、转辙机、列车占-空闲检测、电源等基础设备组成。

5.2.2 车指挥控制系统可与综合监控系统集成，构建以车为核的综合动化系统。

6.2.3 车指挥控制系统中级设备可纳云平台系统由其统部署，纳云平台系统后不应降低车指挥控制系统的安全完整性等级。

6.2.4 联锁系统可与ATP系统合并设置。

6.2.5 信号系统按地域划分，可分为控制中心设备、车站及轨旁设备、车辆基地设备、车载设备、维修中心设备、试车线设备，可根据需要设置备用控制中心或培训中心设备。

# 6.3 闭塞方式

# 6.3.1一般要求

6.3.1.1闭塞式应满足车密度、运速度和交路等运营组织需求。  
6.3.1.2 闭塞式应能实现列车在系统最不利条件下以规定的安全间隔运，其列车安全间隔应满足动实施强迫停车时的最大停车距离及司机控制列车停车时的最大停车距离。  
6.3.1.3 应采空间间隔的闭塞式，包括绝对制动距离和相对制动距离。  
6.3.1.4 城市轨道交通的闭塞制式可采用固定闭塞、准移动闭塞和移动闭塞。固定闭塞可采用进路闭塞和站间闭塞方式。  
6.3.1.5 与衔接的城市轨道交通线路间的闭塞式，可采用半动闭塞、电话闭塞等式。  
6.3.1.6移动闭塞制式互联互通线路间的联络线宜采用移动闭塞制式。

# 6.3.2技术要求

6.3.2.1 闭塞分区的划分或列车最安全间隔应根据列车运密度、线路条件、车辆特性及信号系统的列车控制模式、限速等级等条件设置，并应通过列车运模拟确定。不同闭塞制式系统应符合下列要求。

a) 固定闭塞应以固定的闭塞分区作为追踪列车间的安全间隔。列车最安全间隔应根据车间隔、列车长度、列车限速等级、最不利条件下的制动距离等因素确定，并应根据列车控制式确定保护区段的设置。  
b) 准移动闭塞的列车最安全间隔应以前列车所在闭塞分区端为安全防护点，由后续列车以当前速度制动停车所需行距离加安全保护距离确定。  
c) 移动闭塞的列车最小安全间隔可按前方列车骤然停车，由后续列车以当前速度制动停车所需距离加安全保护距离确定。也可按前列车停车后的安全位置为标点，由后续列车以前后两车制动距离的差值加安全保护距离确定。

6.3.2.2 固定闭塞系统地面设备应向车载设备传递速度码或其他车载设备所需信息；准移动闭塞系统地面设备应向车载设备提供所需信息，车载设备应向地面设备发送开关门信息；移动闭塞系统应实现列车运控制所需的车地或车车连续双向信息传输。

6.3.2.3 准移动闭塞、移动闭塞应采连续式速度-距离模式曲线控制式。

6.3.2.4 固定闭塞、准移动闭塞复线区段宜采单向闭塞式。移动闭塞复线区段宜采用双向闭塞式。单线双向运的区段，应采用双向闭塞式。

# 6.4 系统技术平等级

# 6.4.1 等级划分

系统技术平等级可按列车运动化技术平、车指挥控制技术平、闭塞制式技术平和系统控制等级进分级。

# 6.4.2 列车运行自动化等级

6.4.2.1 列车运行自动化等级的分级应源于给定的列车运行基本功能在运营人员和系统之间的责任分配。

6.4.2.2 列车运自动化等级应分为：a）GOAO:目视行车；b） GOA1:非动列车运；

c） GOA2:半动列车运；  
d） GOA3:有值守列车动运；  
e） GOA4:值守列车动运。

6.4.2.3 高列车运动化等级系统应能兼容低列车动化等级需求。根据系统运用情况，结合运营要求，列车运行自动化等级应能进行转换。

# 6.4.3 行车指挥控制等级

按采的车指挥控制技术平分级，可分为列车运监控级和列车动监控级。

# 6.4.4 闭塞制式等级

按采用的闭塞制式技术平分级，可分为固定闭塞级、准移动闭塞级和移动闭塞级。

# 6.4.5 系统控制等级

按采用的系统控制等级分级，可分为连续式控制级、点式控制级和联锁控制级。

# 6.5驾驶模式

6.5.1系统可采用的列车驾驶模式包括FAM、CAM、AM、CM、RM、RRM、EUM，且应符合下列规定：

a) FAM模式应实现在动化区域内，在系统防护下司乘人员预的列车全动运；  
b) CAM模式应实现在自动化区域内，经人工远程操作确认后，在系统防护下司乘人员干预的列车全自动限速运行；  
c) AM模式应实现在ATP子系统的安全防护下司机监控列车自动运；  
d) CM模式应实现在ATP子系统的安全防护下司机驾驶列车运；  
e) RM模式应在ATP子系统的防护下，实现人工驾驶列车按规定限速运，当列车超速时，ATP设备应实施紧急制动控制列车停车；  
f) RRM模式应能实现在动化区域内，经人远程操作确认后，列车在系统防护下按规定限速在移动授权范围内运行，当列车超速时，ATP设备应实施紧急制动控制列车停车；  
g) 当采EUM模式时，系统不控制列车运，应由司机按操作规程驾驶列车运。

6.5.5 根据信号系统的性能特点和运营需要，驾驶模式可动或转换。列车驾驶模式转换应符合下列规定：

a） 特定运营需要时，经确认应安全并授权后，允许列车以CAM、RRM、RM或EUM等驾驶模式运行；b） 当配置点式降级控制系统时，连续式控制级的AM、CM驾驶模式应能与点式系统相兼容。

# 6.6 信号显示

6.6.1 信号显应分为地信号显和车载信号显。

6.6.2 信号显示应准确、清晰，且应符合下列规定：

a） 车载信号显与地信号显应意义致或含义相符；  
b） 当地信号为主体信号且其信号显熄灭或显意义不明时，应视为禁信号；  
c) 当车载信号为主体信号时，正线地面信号宜灭灯；  
d) 相同的系统控制等级下，信号显示含义应唯一。

6.6.3 信号机的设置应满车组织和安全防护要求。

6.6.4 信号机显示距离应符合设计要求。当信号显示距离不满时，以固定闭塞制式为主系统的正线线路可设置复示信号机，其他制式线路不宜设置复示信号机，可设置接近预告牌。

6.6.5车载信号应正确表地设备赋车载设备的信息。

6.6.6 车载信号的显示可根据车载设备功能、系统构成特点具有不同的表示式和内容。车载信号宜有列车实际速度、标速度、目标距离、列车超速及设备故障等声光报警、表示。

# 7 子系统要求

# 7.1 行车指挥控制

# 7.1.1 一般要求

7.1.1.1 应能满足车组织的需要、提动化管理平、减轻作员的劳动强度并便操作。车指挥分为调度和计算机辅助调度。

7.1.1.2 行车指挥控制子系统技术实现方式包括ATS和CTC两种系统技术。宜采用ATS，也可根据工程需要采用CTC。

# 7.1.2 列车自动监控

# 7.1.2.1 功能要求

7.1.2.1.1应具有列车自动识别、列车运自动跟踪和显示功能。  
7.1.2.1.2 应具有运时刻表或运图的编制及管理功能，可具备运图的实时编制及调整功能。  
7.1.2.1.3 应具有列车运计划及车辆管理功能。  
7.1.2.1.4 应具有进路动和排列功能。  
7.1.2.1.5 应具有临时限速设置和取消功能。  
7.1.2.1.6应具有列车运调整功能，应具备扣车、跳停及提前发车等功能。  
7.1.2.1.7应具有控制权转换功能。  
7.1.2.1.8 应具有列车运实迹、运营操作、运营数据的记录以及各种记录的回放、输出及统计处理  
功能。  
7.1.2.1.9 应具有系统故障处理和故障恢复处理的功能。  
7.1.2.1.10 可具有模拟演示及培训功能。  
7.1.2.1.11应具备与播、PIS、专线、时钟、综合监控等系统接的功能。  
7.1.2.1.12FAO线路ATS子系统除应具备 $7 . 1 . 2 . 1 . 1 { \sim } 7 . 1 . 2 . 1 . 1 1$ 的功能外，还应具备下列功能：

a) 自动控制区域、非自动控制区域分区显示功能；  
b) 宜具有通过人工操作实现对列车设置全自动运授权的功能；  
c) 当与其他系统联动控制时，应实现全自动运行正常、故障及应急场景的管理和控制功能；  
d) 根据运营场景，中心调度员应能实现远程控制功能。

# 7.1.2.2 技术要求

7.1.2.2.1 监控范围应按线路、站场确定的建设规模、运用要求和车组织确定，监控能力应与线路远期条件相适应。  
7.1.2.2.2 可监控条或多条运营线路。当监控多条运营线路时，各条线路应具有独运营或混合运营能力。  
7.1.2.2.3应由控制中心、车站和车辆基地设备等组成，主要设备应采用冗余技术，主、备设备应实现无扰自动切换。  
7.1.2.2.4 控制中应配置车指挥专调度作站、显示终端。车站ATS作站与联锁控制作站宜合并设置。可配置列车运实迹记录设备。  
7.1.2.2.5设置备用控制中心的线路，主控制中心与备用控制中心可互为热备。  
7.1.2.2.6应能降级运用，应能减少故障影响范围。

7.1.2.2.7 宜具备多级控制能，可具有下列控制等级：

a) 中心自动控制；  
b) 中心调度员人工控制；  
c) 车站、车辆基地自动控制；  
d) 车站、车辆基地值班员人工控制。

7.1.2.2.8 控制应优先于动控制。

7.1.2.2.9 在控制权转换过程中，不应影响设备功能执和列车运。  
7.1.2.2.10应能实现实时、连续的正确显示和控制。  
7.1.2.2.11应满足列车运交路的需要，并应根据运时刻表、列车识别号和联锁表所规定的进路等条件，实现列车进路动控制及人工操作。  
7.1.2.2.12在具有折返条件的车站应能动办理折返进路。  
7.1.2.2.13 当列车实际运与计划偏差在预定的容忍偏差范围内时，系统宜能动调整列车运。7.1.2.2.14系统可结合客流变化、在线设备的运行状态、人工指令等信息，计算最优的调整策略，实现列车运自动调整或为人工调整提供辅助。  
7.1.2.2.15 区间运调整可采时间调整式或运等级调整式。互联互通线路区间运调整宜采用时间调整方式，当采用运等级调整方式时，运等级定义及调整效果应一致。  
7.1.2.2.16具有灵活编组功能的线路，系统应能人工或自动控制列车联挂或解编，并应对列车联挂或

解编的状态进监督。

7.1.2.2.17 互联互通线路控制中心级设备宜支持与线网中心设备、其他线路中心设备接口，实现本线与其他线列车运表示、车计划管理、相连接区域信号状态显示和线间车信息的互传。  
7.1.2.2.18 发车计时器应能按系统给定的站停时间指列车的发车倒计时时间、发车时刻、晚点时间信息。跳停、扣车作业应有特殊表示。  
7.1.2.2.19 针对跳停、多交路运及其他特殊运营组织需求，ATS系统应能向PIS、播、车辆等系统提供其用以实现乘客服务所需的信息。

# 7.2 列车动防护

# 7.2.1 功能要求

7.2.1.1应具有列车位置检测、列车安全间隔控制功能。  
7.2.1.2 应具有监督列车运速度、列车超速防护控制功能。  
7.2.1.3 应具有列车运向监督、预期移动防护和退防护功能。  
7.2.1.4 应具有为列车车门、站台屏蔽门的开启提供安全监控的功能。  
7.2.1.5 应具有列车车门状态、列车停稳状态和列车完整性状态监督功能。  
7.2.1.6 当采车轮转速测定列车速度时，应具有列车空转打滑检测和轮径校准功能。  
7.2.1.7 当具备连续车地数据通信功能时，应实现临时限速防护、区域防护、站台屏蔽门状态监督及防  
护功能，可具备防淹门状态监督及防护功能。  
7.2.1.8交流牵引供电线路可具备过分相控制功能。  
7.2.1.9 在直交流牵引供电制式转换区域，可持实现牵引供电制式动转换控制功能。  
7.2.1.10 车载设备应具备记录司机操作和设备运状况的功能。当具有车地线通信功能时，车载运  
志宜具备远程或动下载功能。

7.2.1.11 FAO线路ATP系统除应具备 $7 . 2 . 1 . 1 \sim 7 . 2 . 1 . 1 0$ 的功能外，还应具备下列功能：

a) 远程及本地休眠、唤醒功能；  
b) 人员防护功能；  
c) 障碍物防护功能；  
d) 发车安全防护功能；  
e) 冗余测速及定位功能；  
f) 对自动对位调整、蠕动运、远程限速运的安全防护功能；  
g) 当车辆基地内设置停车列检库及洗车库库门时，应具备库门防护功能；  
h) 车门与站台屏蔽门间的对位隔离接口联动功能；i） 执或转发ATS发送的远程控制命令的功能；  
j） 本地和远程紧急制动缓解功能。7.2.2 技术要求  
7.2.2.1 应由地设备和车载设备组成。  
7.2.2.2 应符合故障-安全原则，并应采必要的冗余和安全技术。  
7.2.2.3 车载设备应适合列车的应环境与运条件，应稳定可靠，并应具有抗扰的能。  
7.2.2.4 封闭线路的运营列车均应装备ATP车载设备，列车尾两端宜各设套ATP车载设备。7.2.2.5 应在车辆基地出线、正线、折返线、渡线、停车线、车辆基地动化区域范围内提供列车运安全防护控制。  
7.2.2.6应持多种运等级和驾驶模式，应具有多种运等级列车混合运能。  
7.2.2.7 具有灵活编组功能的线路，系统应能对列车联挂或解编作业安全防护，对联挂及解编状态进监督。  
7.2.2.8 应以导致列车停车为最安全准则。当执强迫停车控制时，应切断列车牵引。当施加紧急制动时，列车停车过程不应中途缓解。FAO线路通过远程施加紧急制动停车后，若导致列车紧急制动的原因已消除，则紧急制动宜能通过远程操作缓解。  
7.2.2.9 车地或车车间连续通信中断或地信息严重丢失、列车完整性丢失、列车超速、列车预期移动、车载设备重度故障等应导致紧急制动，且宜在车载人机界面或调度工作站上显示紧急制动引发的原因。  
7.2.2.10 对于FAO线路，当ATP实施紧急制动停车且紧急制动缓解后再次启动列车时，应经确认方能进入AM驾驶模式；对于FAO线路，当ATP施加的不可缓解紧急制动解除后再次启动列车时，应经人工确认方能进FAM驾驶模式。  
7.2.2.11 宜采连续式信息传输、连续式速度控制式;宜采连续式速度-距离模式曲线。  
7.2.2.12 在安全防护预定停车点的外方应设安全防护距离或保护区段，安全防护距离及保护区段长度应通过计算确定。  
7.2.2.13 ATP地设备向ATP车载设备传送的允许速度指令、标速度、标距离、站台屏蔽状态等信息，应满ATP车载设备控制式和控制精度的需要。  
7.2.2.14 装备ATP系统的列车，其实际车速的表应由ATP系统的车载设备驱动。  
7.2.2.15 测速装置宜采冗余技术。当不具备独安装车轮速度传感器条件时，可通过与车辆接获取速度信息，也可采用其他测速技术。  
7.2.2.16ATP车载设备应具有必要的显、响报警和故障记录装置。  
7.2.2.17车载信号设备检宜通过车载设备检完成。  
7.2.2.18 固定闭塞系统、准移动闭塞系统应实现闭塞分区的列车占-空闲的检测，并应以闭塞分区为单元检测列车的位置。移动闭塞系统应通过列车定位技术及车地信息的双向传输实现列车的准确定位。  
7.2.2.19 列车占-空闲检测可采轨道电路、轨旁环线、计轴等式实现。  
7.2.2.20 列车定位技术可采轨道电路、轨旁环线、应答器、线、卫星定位或辅以测速设备或测速信息等式实现。  
7.2.2.21 列车位置检测设备在规定的作环境、线路状态和电磁扰影响等条件下，应安全和可靠地实现列车位置的检测，不应错误地出现升级信息。  
7.2.2.22 列车位置检测设备应具有差错控制能，应防因地设备故障、信息丢失导致误控或失控，同时应能连续监视信息通道状态，当信息传递中断时，应采取安全措施。  
7.2.2.23 采交流牵引供电的线路，信号系统可根据需要向车辆提供过分相控制的使能及指令。

7.2.2.24 存在直交流牵引供电制式转换需求的线路，信号系统可根据需要向车辆提供转换控制的使能及指令。

# 7.3 列车自动运

# 7.3.1 功能要求

7.3.1.1应动或启动列车并实现站间动运。  
7.3.1.2 应控制列车实现车站定点停车、车站通过和有司机监督或司机监督的折返作业。  
7.3.1.3与车指挥控制系统相结合，应实现列车运动调整。  
7.3.1.4 应具有车门和站台屏蔽门开闭的动控制功能。  
7.3.1.5 应具有列车运行节能控制功能。  
7.3.1.6 FAO线路ATO子系统除应具备 $7 . 3 . 1 . 1 { \sim } 7 . 3 . 1 . 5$ 的功能外，还应具备下列功能：a) 远程和本地唤醒、休眠功能；b) 全自动进出车辆基地、自动对位调整、自动站台发车、全自动折返功能；c) 清客功能；d) 配合车门与站台屏蔽门实现对位隔离功能；e) 可实现蠕动运行模式、远程限速运行功能；f) 可实现自动洗车功能。

# 7.3.2 技术要求

7.3.2.1 应在ATP系统防护下实现动运功能。  
7.3.2.2 应能控制列车实现牵引、制动、惰、匀速运等多种运况，并满不同车间隔的运要求和列车运行调整的需要。  
7.3.2.3 列车停车控制过程应满舒适度、快捷性和停车精度的要求。  
7.3.2.4 ATO系统不应同时输出制动指令和牵引指令。  
7.3.2.5 当ATO系统发故障时，应能给出报警提，并能转为控制。  
7.3.2.6 列车在区间动停车后，当条件具备时，应能控制列车动启动。对于FAO线路的车站发车，列车启动宜由司机控制，当列车动启动时，应满相应的安全防护条件。

7.3.2.7 FAO线路ATO系统除应符合 $7 . 3 . 2 . 1 \sim 7 . 3 . 2 . 6$ 的要求外，还应符合下列要求：

a) 应采用必要的冗余技术；  
b) ATO车载设备应根据列车运动化等级配置休眠唤醒设备；  
c) 当列车在站台停车后，停车位置未在精确停车范围内且未超出允许跳跃的范围时，ATO子系统应能控制列车跳跃，完成自动对标精确停车；  
d) 车载休眠唤醒单元应与车辆联动，实现远程或本地的休眠或唤醒操作，并应实时向控制中汇报列车休眠或唤醒状态及报警信息；  
e) 可与车辆联动，控制列车按规定的洗车流程完成自动洗车。

# 7.4联锁

# 7.4.1 功能要求

7.4.1.1应能按一定程序和条件控制道岔、信号，建立进路。7.4.1.2 应实现与ATP和车指挥控制子系统的结合，实现进路的人工或动控制。7.4.1.3应实现对列车进路、引导进路、调车进路的控制功能，应实现进路的建立和解锁、信号关闭和开放、道岔操纵及锁闭、信号机封锁和解封、道岔封锁和解封、区段封锁和解封，可具备临时限速功能、车站

扣车功能。

7.4.1.4 应实现办理与系统控制等级相对应的列车进路，并应能办理不同系统控制等级的混合进路，持不同控制等级列车混合运。  
7.4.1.5 单轨、中低速磁浮、动导向轨道线路联锁系统应具有与道岔控制设备接口的功能，应实现道岔联锁集中控制功能。  
7.4.1.6 应实现区段占用状态、进路状态、信号开放状态、道岔状态等各种表示的显示以及故障或异常情况下发出的声光报警。  
7.4.1.7 应具有与站台紧急关闭按钮、站台屏蔽门的接和设备监控功能，可具备与防淹门接的功能。FAO线路应具有与人员防护开关的接口和设备监控功能，可具有与洗车机、车库门、门禁接口的功能。

# 7.4.2 技术要求

7.4.2.1可分为继电联锁和计算机联锁，宜采计算机联锁。  
7.4.2.2 应符合故障-安全原则，应采必要的冗余和安全技术。  
7.4.2.3当计算机联锁采用电子执行单元时，电子执行单元宜冗余配置，电子执行单元与联锁计算机之间应采安全冗余通信通道。  
7.4.2.4联锁子系统逻辑控制设备可与ATP设备合并设置，并由ATP设备实现联锁的功能。7.4.2.5可通过联锁系统实现基于进路闭塞或站间闭塞的车式。  
7.4.2.6 进路上道岔、信号机和区段的联锁关系应正确，当联锁条件不符时，不应使进路开通、信号开放。  
7.4.2.7敌对进路应相互照查，不应同时开通。  
7.4.2.8 应能办理列车和调车进路，可实现车站有关进路、端站折返进路的动排列。  
7.4.2.9 具有灵活编组功能的线路，系统应能实现对列车联挂或解编作业的进路控制。  
7.4.2.10 联锁道岔应能实现进路锁闭、区段锁闭及人锁闭，应能实单独操纵和进路选动;影响车效率的联动道岔宜采同时启动式。  
7.4.2.11 进路锁闭应分为预先锁闭和接近锁闭。进路解锁应分为进路一次性解锁方式和逐段解锁方式。锁闭的进路可随列车运行自动解锁或人工办理取消进路解锁，并应防止错误解锁。当接近锁闭时，取消进路应延时解锁或收到停车保证信息后即解锁。延时解锁时间应保证列车能在进路解锁前安全停车。  
7.4.2.12 进路办理宜采进路的始终端控制式，也可采其他进路控制式。当联锁设备与车指挥控制设备结合时，应能随列车运自动排列进路。联锁设备应根据ATP系统的要求设置相应的保护进路或保护区段。  
7.4.2.13涉及车安全的紧急控制应由车站办理。  
7.4.2.14 当装设引导信号的信号机因故不能正常开放时，可使用引导进路锁闭式开放引导信号。7.4.2.15 计算机联锁子系统设备的操纵应采用屏幕显示器与鼠标器的组合设备，车辆基地可采用按钮控制台或其他操纵式。  
7.4.2.16 在车地数据通信正常的情况下，联锁系统设备宜能提供车载信号设备所需的信息。7.4.2.17 车站站台公共区及车站控制室应设站台紧急关闭按钮，FAO线路应设置员防护开关，可在站台设置开门联动按钮、关门联动按钮、清客确认按钮。  
7.4.2.18涉及行车安全的、与联锁结合的控制电路均应符合故障-安全原则。  
7.4.2.19 当单轨、中低速磁浮、动导向轨道线路的道岔具备联锁集中控制、现地控制和应急控制多种控制模式时，联锁子系统应具有控制模式状态表示。当由联锁集中控制模式转换为现地控制模式时，应取得信号系统授权。当道岔处于人工现地模式和人工应急控制模式时，联锁子系统不应

操纵道岔和办理经过该道岔的进路。

# 7.5数据通信

# 7.5.1功能要求

7.5.1.1 应实现信号系统系统间、系统内部以及与外部接设备间安全可靠的信息交换。7.5.1.2 应制定相应的安全防护策略，同时应具备多种式和层次的访问控制安全机制，且应满信息安全防护等级的要求。  
7.5.1.3 应具备数据通信络管理功能。

# 7.5.2 技术要求

7.5.2.1应由有线络、线通信络或车地通信设备和络管理设备组成。

'.5.2.2 有线络、线通信络应采冗余设计，单点故障不应影响信息传输的实时性和连续性。

7.5.2.3 地面有线络可采独的通信网络，与其他系统的接应有隔离措施。信号系统内部各系统应采用相互独的传输通道，传输通道的划分可采用物理通道方式或者逻辑通道式，单个传输通道的故障不应影响其他传输通道的正常作。车载络可与车辆络融合设置，融合络应能满信号系统的安全完整性等级的要求。

7.5.2.4 数据通信系统容量、传输速率和传输距离应满控制列车运实时监控的需要和车指挥的运用要求。

7.5.2.5 应满各系统或设备对信息传输速率和信息传输质量的要求，应降低信息传输的损耗、延时、丢包率、误码率，满信号系统可靠性、可用性的要求。在列车最速度运时的漫游切换，不应影响数据通信的连续性。

7.5.2.6 数据通信设备在规定的作环境、线路状态和电磁干扰影响等条件下，应实现信息可靠传输。

7.5.2.7 线通信络或车地通信设备可利轨道电路、轨旁环线、卫星通信、应答器、线通信等信息传输式。

7.5.2.8 信号系统的信息安全防护应满GB/T22239的要求，并应符合下列规定。

a) 信号系统的信息安全防护应以保障信号系统业务功能安全为首要目标，所采取的安全措施，包括相应的安全设置和部署的安全产品，不应影响信号系统自的可靠性和安全性。  
b) 信息安全设计不应改变信号系统结构，不应影响信号系统数据传输实时性、丢包率；宜采并联型防护设备，当设置串联型防护设备时，应采取冗余措施。  
c) 应具备信号系统边界的隔离防护功能，宜具备安全审计、身份鉴别、漏洞扫描、主机防护等功能。

# 7.6 维护监测

# 7.6.1 功能要求

7.6.1.1应能与车指挥控制子系统、ATP子系统、联锁子系统、数据通信子系统、电源等设备接口，宜实现完整统的信号系统设备维护监测功能。  
7.6.1.2应具备监测采集、故障诊断、综合分析、报警集中、维护监测子系统自诊断和查询与回放功能，宜具有设备健康状态趋势诊断功能及维护引导功能，可具备三维显示功能。  
7.6.1.3 应具备就地和远程的监测、预警、报警功能，应具有中级集中监测告警功能。  
7.6.1.4 应能在维护监测作站实施远程设备状态监测、故障集中报警、故障诊断定位和维护管理功能。  
7.6.1.5 应具备故障分级分类报警功能，对涉及车安全或直接影响车的报警应具有声提。

7.6.1.6应能对操作人员所进的操作、时间、对象、内容、结果等信息进记录，应能在维护监测子系统的维护作站实现对整个信号系统历史数据回放。7.6.1.7维护监测系统宜具备与线维护系统或其他数据平台接的功能。

# 7.6.2 技术要求

7.6.2.1维修中服务器宜采用冗余结构，应实时监测信号系统在线运状态。  
7.6.2.2 正线车站、车辆基地可设置信号集中监测设备对基础信号设备、电源设备的运状态、报警信  
息等进行监测和采集。  
7.6.2.3 当维护监测系统具有三维显示功能时，应实现设备三维场景的重建和作状态的实时复现。  
7.6.2.4维护监测系统可与其他系统整合构建多专业综合维护系统。

# 8 车辆基地信号系统

8.1 车辆基地信号系统可采用调车进路控制或列车进路控制。  
8.2 车辆基地应全部纳车指挥控制系统监视范围，纳车指挥控制系统监控或监视的车辆基地应实现车辆基地内列车车组号的追踪显。  
8.3 车辆基地应根据列车运动化等级设置列车定位设备、ATP设备、ATO设备及区域防护设备。8.4 车辆基地与试车线的接设计应满试车作业与车辆基地作业互不影响的要求。  
8.5 试车线信号地设备的布置应满车载信号设备动态测试和双向试车的需要，其地设备应与正线信号设备致。

8.6 试车线与正线的车地线通信设备间不应相互扰。

8.7 动化车辆基地除应符合 $8 . 1 \sim 8 . 6$ 的要求外，还应符合下列规定：

a） 车辆基地动化区域与动化区域之间应严格分区，并应设置动化区域和动化区域转换区满足驾驶模式转换需要；  
b) 车辆基地停车列检线及与其连接的咽喉区应设置为自动化区域；  
c）自动化区域内应具有ATP防护功能，可具备ATO功能。

8.8 全动运车辆基地除应符合8.7的要求外，还应符合下列规定：

a) 自动化区域应设置人员防护开关防护自动化区域内的人作业；  
b) 车辆段洗车线宜设置为自动化区域；  
c) 车辆基地动作业宜包括列车唤醒、库内发车、自动化区域内运、回库、洗车、动转线、清扫、休眠等作业。

# 9人机界面

9.1人机界面应整体美观、协调、满足人体工程学的要求，并应符合GB/T16251的规定。  
9.2 人机界面的显信息应清晰明确，并应具备简洁、直观、便、明确的人机对话式，且应易于监控、识别和操作。  
9.3 机界应具有显功能、语播报功能、通信功能、键盘输功能、亮度调节、量调节功能和记录功能。  
9.4 机界应具备登录权限管理、控制权限管理功能。  
9.5 站场图的显应与实际站场相致，显状态应与室外设备实际状态相致，且种显应仅对应种设备状态或表达种信息。  
9.6 调度相关作站应布局合理，且应采简体中界面的图形户接式;应具备多级菜单、多级

窗、图形移动和图形缩放的功能，并应能持单屏或多屏幕显。

9.7 对由确保安全的操作命令，应有相应的安全操作段和操作记录。机界上显的各种记录、故障及报警信息应意思明确，并应便于维修人员跟踪记录、查找故障。

9.8 每个驾驶室中均应配置车载信号显屏，根据需要可与车辆显屏合并设置，且应满下列要求。

a) 显示屏的亮度可手动或自动调整。当自动调整时，屏幕的亮度应根据周围环境的变化而自动调节，应能使显示屏保持可见状态。  
b) 车载信号显屏显可根据车载设备功能、系统构成特点具有不同的表式和内容。  
c) FAO线路车载信号显屏宜具有发车计时器显功能。  
d) 互联互通线路车载人机界面显示元素、显示含义宜统，车载报警信息显及含义宜统一。

# 10电源要求

10.1 信号系统供电应为一级负荷，且应设两路独立电源。  
10.2 信号系统应采集中电源和分路馈电式，其交直流电源应对地绝缘。  
10.3 当电源电压波动超过用电设备正常工作范围时，应采取稳压和滤波等措施。  
10.4 车载信号设备电源应由车辆提供直流电源或经变流设备供电，并应设过压和过流保护。10.5 信号系统应采专的电源屏供电。当采UPS整合供电时，电源屏应具有主副电源动和手动切换装置，切换时不应影响用电设备正常工作。  
10.6 信号系统电源设备应具备不间断供电功能或由UPS整合专业提供不间断供电功能。当双路外电源供电失效时，不间断供电功能应满为控制中、车辆基地、维修中、车站的信号系统设备提供不小于 $3 0 \ \mathrm { m i n }$ 的供电要求。  
10.7电源容量除应满最负荷需要外，且应留有备用容量，备容量不宜于 $30 \%$   
10.8 电源设备主要功能单元宜采模块化结构，直流供电模块应采取冗余措施。  
10.9 电源设备各功能模块应采带电插拔、鉴别防错连接技术。  
10.10 电源设备宜实现对各模块及主要元器件、电池的作状态及数据进实时在线监测。电源系统的维护和监测信息宜纳维护监测系统统管理。  
10.11电源屏输出室外的设备交流供电回路应采隔离供电式。

# 11接口

11.1 信号系统接可分为系统内部各设备间、系统与系统间的内部接及信号系统与其他专业系统的外部接。

11.2 信号系统内部接可包括：

a) 同一子系统设备间的接口；  
b) 行车指挥控制子系统的设备与联锁子系统、ATP子系统设备的接口；  
c) 联锁子系统与ATP子系统设备的接口；  
d) 当由信号系统设置车地和车车数据通信设备时，其与地面设备、车载设备的接；  
e) 维护监测子系统与其他各子系统的接口；  
f) 车载信号设备内部接可包括ATO系统与ATP系统的接；  
g) 联锁设备与现场设备的接口，现场设备可包括信号机、转辙机、紧急关闭按钮、自动折返按钮站台门控箱、人员防护开关等。

11.3 信号系统内部接可分为安全性接和安全性接。联锁设备与ATP设备的接、联锁设备与现场设备的接口、联锁设备与联锁设备的接口、ATP地面设备与ATP地面设备的接口、ATP地面设备与ATP车载设备或两车ATP车载设备间的接应为安全性接。

11.4 信号系统与外部接可包括：

a） 车辆、通信、供电、站台屏蔽门、综合监控系统、环境与设备监控、防淹门的接口；  
b） FAO线路宜包括与车库门、洗车机、站台间隙探测、障碍物检测系统的接，可包括与门禁系统接口；  
c) 对设置各系统综合数据平台的工程，可具备与综合数据平台的接口；  
d） 单轨系统、中低速磁浮系统线路应具备与道岔系统设备接。

11.5 信号系统外部接口可分为开关量接、串接、网络接以及模拟量接等。ATP或ATO车载设备与车辆相关设备涉及安全功能的接口应为安全性接口。

11.6 信号系统与外部接处应设置可靠的络安全隔离措施。接口宜使标准、通的通信协议。

11.7 车载信号设备与车辆接口电路的布线应与其主回路等环节的压布线分开敷设并实施防护，与车辆电气的接口应有隔离措施。

# 12可靠性、可性、可维修性和安全性（RAMS）要求

# 12.1可靠性

12.1.1 信号系统及其产品应进可靠性描述。  
12.1.2 应采可靠性的元器件。  
12.1.3 应采用必要的冗余技术。  
12.1.4 应减少可能中断系统运的单点故障。  
12.1.5 应提供“带故障运”的能。  
12.1.6 系统可靠性参数应通过硬件可靠性计算得出，应采公式(1)计算：

$$
\mathrm { M T B F } = \frac { 1 } { \lambda }
$$

……(1)

式中：

λ———故障率。

# 12.2可用性

12.2.1 信号系统应进可性描述。  
12.2.2 重要信号设备应有冗余措施。  
12.2.3 信号设备应具有必要的作状态检测、故障诊断和报警等技术措施。  
12.2.4 信号设备应有故障指。  
12.2.5系统可用性参数应通过计算估算得出，应采用公式(2)计算：

$$
A = \frac { \mathrm { \Delta M T B F } } { \mathrm { \Delta M T B F + M T T R } } \times 1 0 0 \%
$$

……(2)

式中：

$A$ ——可性。

# 12.3 可维修性

12.3.1 信号系统及其设备应具有调整少、维修量、可预防性维护及故障维护的技术手段，并应具有测试点、故障隔离及诊断措施，应减少设备修复时间、降低维护成本。

12.3.2 应制定合理的维护、更换策略，并应配置在线维护持设备减少信号设备故障及停机时间。

12.3.3信号设备宜有状态修维护的能，便于维修、更换。

# 12.4 安全性

12.4.1 信号系统的安全性应符合下列规定：

a）涉及车安全的设备应采成熟、经过运实践并证实安全可靠的设备；

b） 安全系统或设备的安全功能及安全完整性等级应提供相关件予以证实，件证实的内容应包括：

1） 安全系统或设备的研发程序及安全管理组织体系应符合GB/T28808、GB/T28809和  
GB/T21562的规定；  
2) 应已实施危险识别和分类、危险处理和评估；  
3) 对信号系统、子系统、设备的安全功能应分析和确认；  
4) 应完成故障模式及故障影响范围的确认；  
5) 应完成了具有外界扰的系统运试验；  
6) 应具有安全功能检测报告；  
7) 应根据需要提供安全性试验证明。

12.4.2 安全性要求及安全完整性等级应在信号系统、设备研发或技术规格中明确描述，其安全完整性等级应分为下列4级：

a）1级： $1 0 ^ { - 6 } \ \mathrm { h } ^ { - 1 } { \leqslant } \mathrm { T F F R } { < } 1 0 ^ { - 5 } \ \mathrm { h } ^ { - 1 }$ ;  
b) 2级： $1 0 ^ { - \tau } \mathrm { \Delta h ^ { - 1 } { \leqslant } T F F R { < } } 1 0 ^ { - 6 } \mathrm { \Delta h ^ { - 1 } }$ ;  
c） 3级： $1 0 ^ { - 8 } \ \mathrm { h } ^ { - 1 } { \leqslant } \mathrm { T F F R } { < } 1 0 ^ { - 7 } \ \mathrm { h } ^ { - 1 }$ ;  
d) 4级 $1 0 ^ { - 9 } \ \mathrm { h } ^ { - 1 } { \leqslant } \mathrm { T F F R } { < } 1 0 ^ { - 8 } \ \mathrm { h } ^ { - 1 }$ 。

12.4.3 信号系统或设备在正常和故障情况下，应具有不产导致信号系统不正常运的设计和执失误。依赖于故障-安全设计的功能，不应导致故障不安全模式。在执与安全相关的功能时，各元件所发的故障均应具有显报警功能。

# 13 系统性能

# 13.1系统能力

13.1.1 通则

13.1.1.1 系统能应包括信号系统的监控范围、响应性能、通过能、折返能、扩展能和系统设备故障的降级运用及其复原能力。  
13.1.1.2系统监控范围应结合线路和站场规模设计。  
13.1.1.3 系统应具有良好的实时性，且应按远期线路规模、最在线列车数以及线路最运速度设计，可预留不小于 $30 \%$ 的系统处理能力。

# 13.1.2 监控范围

13.1.2.1 正线信号系统监控范围应包括车辆基地出线、正线区间与车站、停车线和折返线。根据需要可包括车辆基地的部分或全部以及与其他线的联络线。调度区段内的区间、车站应集中监视。13.1.2.2信号系统应能监视或控制调度区内的进路、信号、列车、设备运行、与其他联动系统的状态等，以及从其他线调度区向本线调度区的发车进路、信号及列车等。

# 13.1.3 系统的主要响应性能

13.1.3.1 信息采集的表周期即设备状态变化控制中的显时间不应于2s。

13.1.3.2 控制命令的反应时间即命令发出被控系统开始执的时间不应于 $2 \textrm { s }$   
13.1.3.3 实时控制、各作站及显终端等的操作响应时间不应于1s。  
13.1.3.4 对于列车占-空闲检测设备，列车空闲到占检测的应变时间不应于 $2 \textrm { s }$ ，列车占用到空闲检测的应变时间不应于 $3 \mathrm { ~ s ~ }$ 。  
13.1.3.5计算机联锁设备的处理周期不应于1s。  
13.1.3.6 车载信号设备接收地信息完成处理的时间不应于 $0 . 7 5 \mathrm { ~ s ~ }$ 。  
13.1.3.7 车载信号设备的上电重启时间宜于 $6 0 \mathrm { ~ s ~ }$ 。  
13.1.3.8 当车载信号设备识别到涉及车安全的系统故障时，应即发出紧急制动命令，且其延时不应大于 $0 . 7 5 \mathrm { ~ s ~ }$

13.1.3.9 列车位置及测速应符合下列规定：

a） 列车位置最测量误差不应于 $2 \%$ ;  
b） 于ATP功能的列车测速误差不应于 $3 ~ \mathrm { { k m } / h }$ ;  
c) 用于ATP功能的列车测速分辨率不应大于 $2 ~ \mathrm { k m } / \mathrm { h }$ ;  
d） 零速检测标准应为列车速度不于 $1 ~ \mathrm { k m / h }$ 且持续时间不于 $2 \textrm { s }$ 。

13.1.3.10车载ATO宜采用一次连续控制模式制动，自动控制列车在站台精确停车，停车精度指标应符合下列规定：

a） 停车精度范围 $\pm 0 . 3 \mathrm { ~ m ~ }$ 内的概率不应小于 $9 9 . 9 9 \%$ ;b） 停车精度范围 $\pm 0 . 5 \mathrm { ~ m ~ }$ 内的概率不应小于 $9 9 . 9 9 9 \ 8 \%$ 。13.1.3.11列车纵向冲击率不宜于 $0 . 7 5 ~ \mathrm { m / s ^ { 3 } }$ 。13.1.3.12 地面有线网络信息传输速率不应于 $1 0 0 ~ \mathrm { M b / s } ;$ 信息传输端到端延迟时间不应大于(cid:) $1 5 0 ~ \mathrm { m s }$ 。13.1.3.13 数据通信应采双向愈的环形拓扑结构，当环网中个节点故障时，环愈时间应小于 $5 0 \ \mathrm { m s }$ 。

# 13.1.4 通过能力和折返能力

13.1.4.1系统控制列车的通过能力、折返能力及出入段能力与车站配线形式、线路参数、列车编组、车辆性能、道岔限速及信号系统技术水平等因素密切相关，并应通过列车运行仿真分析计算确定。13.1.4.2通过能应满最车间隔的需求，并应留有余量，余量宜不于 $10 \%$ 。13.1.4.3折返能力应与线路的通过能力相适应，计算的折返能力宜小于最小车间隔时间。13.1.4.4 列车出车辆基地的车能力应与正线列车运能相适应。出车辆基地的列车不应影响正线列车的运行。

# 13.1.5 系统扩展能力要求

13.1.5.1 系统设备硬件及软件应采标准化、模块化配置，应具备系统功能扩展及升级条件。  
13.1.5.2系统控制、设备状态表示及采集的软件与硬件宜具有不于 $30 \%$ 的扩容能力。  
13.1.5.3 与延伸线接轨车站设备配置应预留线路延伸的接及系统能扩展条件。

# 13.1.6系统设备故障的降级运用及其复原能力

13.1.6.1 系统应具有故障降级运用能。系统、系统、设备故障排除后，应具有尽快复原执预定功能的能力。  
13.1.6.2 系统降级情况下的监控范围、响应时间、通过能力、折返能及出车辆基地能力应满列车安全运行和行车组织的需要。

# 13.2 RAMS性能指标

13.2.1 主要系统及设备可靠性指标应符合下列规定：

a） ATS设备的MTBF不应于 $3 . 5 \times 1 0 ^ { 3 } \mathrm { ~ h ~ }$ ;  
b) 计算机外围设备的MTBF不应于 $1 0 ^ { 4 } \mathrm { ~ h ~ }$ ;  
c) ATP/ATO设备的MTBF不应小于 $1 0 ^ { 5 } \mathrm { ~ h ~ }$ ;  
d) 计算机联锁的MTBF不应于 $1 0 ^ { 5 } \mathrm { ~ h ~ }$ ;  
e）线通信设备的MTBF不应于 $5 \times 1 0 ^ { 4 }$ h。

13.2.2 信号系统的可性指标不应于 $9 9 . 9 8 \%$

13.2.3 FAO线路信号系统休眠唤醒设备可性指标不应于 $9 9 . 5 \%$ 。

13.2.4 系统的可维修性指标应符合下列规定：

a) 车载设备的MTTR不应大于 $3 0 \ \mathrm { m i n }$ ;  
b) 控制中心设备的MTTR不应大于 $4 5 \ \mathrm { m i n }$ ;  
c) 车站设备的MTTR不应大于 $4 5 \ \mathrm { m i n }$ ;  
d) 轨旁设备的MTTR不应大于 $4 \textrm { h }$ ;  
e) 无线通信设备的MTTR不应于 $3 0 \ \mathrm { m i n }$ 。

13.2.5 主要系统安全功能的安全完整性等级指标应符合下列规定：

a） ATP子系统应达到SIL4；  
b）CI子系统应达到SIL4；  
c) ATO子系统宜达到SIL2;  
d) ATS子系统宜达到SIL2。

# 14 电磁兼容性与防护

# 14.1 电磁兼容性

# 14.1.1通用要求

14.1.1.1 在设计、制造信号设备时，应结合设备的工作电磁环境，电磁干扰不应影响设备的安全性和可靠性；应采用屏蔽、滤波、接地、隔离、平衡以及其他技术措施，应使设备满足其电磁兼容性能要求。14.1.1.2 应消除电磁辐射、感应、传导和静电释放等扰因素对信号设备的正常作产的影响。信号设备、部件也应防对其他系统、部件和运营线路范围内以及附近系统的正常作产电磁扰。14.1.1.3 信号设备在正常工作时向设备外部可能发射的电磁干扰应符合电源和机箱端口试验项目规定的电磁发射限值要求。

# 14.1.2 设备的抗扰度试验

14.1.2.1 信号设备应进抵御外界电磁骚扰能的试验。在设备与外部环境的特定接，包括机箱端口、电源端口、输入输出端口和地线端口上加入标准的电磁骚扰模拟信号，其严酷程度应由试验等级表。设备因外部电磁骚扰影响而使其功能或性能下降的判断依据应采性能判据表示。

14.1.2.2试验项目应包括射频电磁场辐射骚扰、射频场感应的传导骚扰、电快速瞬变脉冲群、浪涌冲击电压、静电放电、工频磁场、脉冲磁场的抗扰度试验。

14.1.2.3 性能判据应包括下列四级：

a）A级：功能或性能正常；  
b） B级：功能或性能暂时降低或丧失，但可恢复；c）C级：功能或性能暂时降低或丧失，需操作者预或系统复位后可恢复；  
d）D级：设备、元件或软件损坏、出错或数据丢失，需经修复处理后可恢复。

14.1.2.4 抗扰度试验应符合下列要求：

a） 抗扰度试验的试验法、试验等级、性能判据应按GB/T24338.4和GB/T24338.5的规定执行；  
b) 对安全设备的性能判据应采用A级；  
c） 对安全设备静电放电抗扰度试验与浪涌(冲击)抗扰度试验可采B级判定。

# 14.1.3设备的电磁骚扰发射试验

信号设备对外界发射的电磁骚扰的试验应包括电源端口的传导发射试验和机箱端的辐射发射试验。试验法和骚扰限值应符合GB/T24338.4和GB/T24338.5的要求。

# 14.2 防护

14.2.1 信号设备正常作时发射的电磁能量不应对周围环境中其他设备或员造成扰和危害，并应防其他设备及车辆等产的电磁骚扰影响信号设备可靠作。  
14.2.2 信号设备与接触或接触轨带电部分之间应留有安全距离。信号电缆线路与强电线路应分开敷设，当二者存在交叉时，宜相互垂直交叉敷设，且在二者间距不满足设计要求时，应采取防护措施。14.2.3 信号属结构物的装设应采取防护牵引迷流对属和隧道体的电蚀措施。  
14.2.4 轨旁设备应防最牵引回流、钢轨不均衡电流的影响。相邻轨旁设备应防作频率的相互串扰。  
14.2.5 装设单轨条轨道电路的车站，相邻轨道电路并联的牵引轨条数应符合轨道电路设备要求。双轨条轨道电路区段的接地引线不应直接引至钢轨。

14.2.6 信号设备的雷击试验应符合TB/T3498的规定。

14.2.7 易受雷电危害的设备应具有雷电感应过电压防护，并应满下列要求：

a) 防护电路应将雷电感应过电压限制在被防护设备的冲击耐压水平以下；  
b) 防护电路不应影响被防护设备的正常工作，且设备受雷电干扰时不应错误动作；  
c) 防雷元器件与被防护设备之间的连接线应最短，防护电路的配线应与其他配线分开，其他设备不应借用防雷元器件的端子；  
d) 室外信号设备、与外线连接的室内信号设备应具有雷电防护措施；  
e) 防雷地线接地电阻应小于 $1 0 \Omega$ 。

14.2.8 信号设备的接地应符合下列规定。

a) 信号设备应设工作地线、保护地线、屏蔽地线和防雷地线等。  
b) 信号设备室应设主接地板，并应通过主接地板接地。  
c) 车载信号设备的地线应经车辆的接地装置接地。  
d) 电源设备及其他带电信号设备的机架、机壳应设保护地线，需工作接地的设备应设工作地线。接地电阻值应符合下列规定：  
1) 保护地电阻应小于 $1 0 \Omega$ ;  
2) 工作地电阻应小于 $4 ~ \Omega$ ;  
3) 信号设备可采用综合接地系统，其接地电阻不应大于 $1 \ \Omega$ 。
